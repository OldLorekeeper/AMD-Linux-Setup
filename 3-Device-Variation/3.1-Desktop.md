## 3.1.4 - Sunshine Configuration

You must manually configure Sunshine to use the automation scripts for HDR, Resolution switching, and GPU Clocks.

**1) Configure Resolution Scripts**

Identify your monitor ID (e.g., DP-1) and target mode numbers:

```
kscreen-doctor -o
```

Edit both the HDR and SDR scripts to match your hardware variables (MONITOR, STREAM_MODE, DEFAULT_MODE):

```
kate ~/Obsidian/AMD-Linux-Setup/5-Resources/Sunshine/sunshine_hdr.zsh ~/Obsidian/AMD-Linux-Setup/5-Resources/Sunshine/sunshine_res.zsh
```

**2) Web UI Setup**

Navigate to the Sunshine Web UI (https://localhost:47990) > Applications.

**Option A: HDR Applications**

Do Command:

```
/usr/local/bin/sunshine_hdr enable
```

```
sudo /usr/local/bin/sunshine-gpu-boost start
```

Undo Command:

```
/usr/local/bin/sunshine_hdr disable
```

```
sudo /usr/local/bin/sunshine-gpu-boost stop
```

**Option B: SDR Applications**

Do Command:

```
/usr/local/bin/sunshine_res enable
```

```
sudo /usr/local/bin/sunshine-gpu-boost start
```

Undo Command:

```
/usr/local/bin/sunshine_res disable
```

```
sudo /usr/local/bin/sunshine-gpu-boost stop
```

## 3.1.6 - Optional: Soularr Stability Management

These steps are for troubleshooting if Soularr breaks after an update.

Run following command to see verbose log for soularr service:

```
sudo journalctl -u soularr.service -f
```

Stay stable:

```
>>> Freeze requirements:
	pip freeze > /opt/soularr/requirements.lock
>>> When Soularr breaks, run:
	sudo pip install --break-system-packages -r /opt/soularr/requirements.lock
>>> Optional quick check:
	cat >/usr/local/bin/check-soularr-deps <<'SH'
	#!/bin/sh
	python - <<'PY'
	import requests, pydantic, schedule
	print("Soularr deps OK")
	PY
	SH
	sudo chmod +x /usr/local/bin/check-soularr-deps
	/usr/local/bin/check-soularr-deps
```
